Cách thu thập & chuẩn hoá dữ liệu

Dữ liệu đi qua pipeline nhiều lớp, mỗi lớp làm sạch thêm.

Nguồn thô                          →  Chuẩn hoá                  →  File sạch              →  API
─────────────────────────────────────────────────────────────────────────────────────────────
Biểu thuế TCHQ (Excel)             →  import + snapshot          →  tax.json (11.871 mã)   →  /api/tax
Cột chính sách (chữ tự do)         →  Gemini bóc tách (AI)       →  tax-enriched.json      →  warnings
Mã văn bản trong chính sách        →  extract-legal-refs         →  legal-docs.json (108)  →  /api/legal-docs
Tiêu đề VB (tra tay, chính thống)  →  legal-doc-titles.json      →  apply-verified-titles  →  tiêu đề thật (30)
Kho tri thức cũ                    →  import-legacy              →  notes/precedents/...   →  /api/notes...
Không bịa dữ liệu — tiêu đề văn bản chỉ ghi khi đã đối chiếu nguồn chính thống (congbao.chinhphu.vn, vbpl.vn, thuvienphapluat.vn…).
Minh bạch tình trạng — văn bản chưa verify để nhãn rõ, mời cộng đồng cùng bổ sung.
Bảo vệ dữ liệu riêng tư — tờ khai khách hàng không bao giờ công khai; trang này chỉ có dữ liệu pháp quy.
Có kiểm thử tự động mỗi lớp dữ liệu trước khi phát hành.

Dùng API thế nào

API dành cho hệ thống (ERP, tích hợp) — cần Bearer token. Xem tài liệu đầy đủ trên GitHub.

Tra thuế 1 mã HS

GET /api/tax?hs=85171300
→ tên hàng, thuế NK/ACFTA/VAT, cảnh báo chính sách + văn bản liên quan.

Tra văn bản pháp luật

GET /api/legal-docs?issuer=BCT
→ danh sách văn bản kèm tiêu đề & link gốc.

Toàn bộ endpoint

Chỉ /api/health không cần token; còn lại cần header Authorization: Bearer <token>.

EndpointMethodTokenCông dụng
/api/healthGETkhôngKiểm tra dịch vụ sống + cấu hình
/api/tax?hs=GETBearerTra thuế NK/ACFTA/VAT + cảnh báo chính sách
/api/search?q=GETBearerTìm mã HS theo từ khoá
/api/kg_chapter?chapter=GETBearerLiệt kê mã HS trong một chương
/api/notes?chapter=GETBearerChú giải chương/nhóm (hỗ trợ GIR)
/api/conflicts?hs=GETBearerCảnh báo các mã dễ nhầm lẫn
/api/precedents?hs=GETBearerTiền lệ phân loại TB-TCHQ
/api/legal-docs · /:codeGETBearerThư viện văn bản pháp luật
/api/products?hs=GETBearerSản phẩm ví dụ cho mã "Loại khác"
/api/customs-types · /recommendGETBearerMã loại hình XNK (QĐ 1357)
/api/suggestPOSTBearerGợi ý mã HS bằng AI + audit GIR
/api/matchPOSTBearer*Sản phẩm ERP đa ngữ → top-K mã HS
/api/describePOSTBearerSinh mô tả khai báo chuẩn TT 39/2018
/api/feedbackPOSTBearerGhi nhận chỉnh sửa của giám đốc
/api/kg_statsGETBearerTổng quan toàn bộ dataset
/api/versions · /version · /version/diffGETBearerPhiên bản biểu thuế + so sánh thay đổi
/api/data-qualityGETBearerBáo cáo chất lượng dữ liệu
/api/admin/overviewGETBearerKPI tổng hợp cho dashboard quản trị

* /api/match có thể bỏ token khi bật HS_MATCH_PUBLIC=true (server-to-server ERP). Về hạ tầng: 12 file function (kịch trần Vercel Hobby), nhiều route đi qua rewrite trong vercel.json (vd dataset.js gánh hàng chục resource) nên URL công khai vẫn gọn.
